name: Comment usage
description: Only add comments for genuinely complex logic; avoid commenting obvious code
type: feedback
---

Use comments sparingly. Only comment complex or non-obvious code.

**Why:** User preference — comments on self-evident code add noise.

**How to apply:** Skip comments on straightforward code (simple assignments, obvious function names, standard patterns). Add comments only when the logic is hard to follow without explanation (e.g., non-obvious algorithms, tricky edge cases, workarounds).
